HtmlSnippet (focused extraction)


{
  "@spec": "https://aionscript.com/specs/sjson/0.1",
  "@type": "HtmlSnippet",
  "@id": "gh:foo/bar/site/index.html@9f8e7d#hero",
  "@context": "https://aionscript.com/contexts/web-doc",
  "@source": "https://github.com/foo/bar/blob/9f8e7d/site/index.html#L1-L40",
  "@timestamp": "2025-01-12T15:32:10Z",
  "@confidence": 0.97,
  "selector": "section.hero",
  "html": "<section class=\"hero\"><h1>Bar</h1><p>Fast web components…</p></section>",
  "text": "Bar — Fast web components…",
  "headings": [{ "level": 1, "text": "Bar" }],
  "links": [],
  "@embedding_model": "text-embedding-3-large",
  "@embedding_dim": 3072,
  "@embedding": [0.05, -0.22, 0.64],
  "@transform": "html_to_text(v1)"
}

Tips

  • Include commit SHA in @source for immutable references.
  • Chunk to ~200–800 tokens; include section/title for stronger retrieval.
  • Set code_language for code blocks; keep both code and summary text.
  • Ship @embedding_model and @embedding_dim with vectors; omit vectors if you store them separately.
  • Add @license if you propagate content; respect repository licenses.

Structured JSON (sJson) Spec v0.1 (Draft)

File extension: .sjson · Proposed media type: application/vnd.aionscript.sjson+json (subject to registration)

Required fields

  • @type — string — logical type or class
  • @id — string | URI — unique identifier
  • @context — URI | object — domain ontology or schema reference

Optional semantic fields

  • @source — URI | string — where the data came from
  • @link — array of URIs or IDs — related objects
  • @confidence — number (0–1) — trust / certainty
  • @embedding — array[number] — vector meaning
  • @timestamp — ISO string — when captured
  • @transform — code string — optional transformation rule

Example


{
  "@type": "UtilityClass",
  "@id": "bg-mint-500",
  "@context": "https://aion.dev/tailwind",
  "description": "Applies mint background color",
  "css": "background-color: var(--color-mint-500);",
  "@embedding": [0.21, -0.34, 0.89],
  "@confidence": 0.98,
  "@source": "github://tailwindlabs/tailwindcss/src/plugins/backgroundColor.js",
  "@link": ["bg-green-500", "bg-teal-500"]
}

Embedded Knowledge

Store embeddings inside .sJson to unify structure and semantics.

{
  "@type": "DocumentChunk",
  "text": "Tailwind’s dark mode now uses @custom-variant",
  "@embedding": [0.02, -0.33, 0.81],
  "@source": "tailwindcss.com/docs/dark-mode"
}

This enables similarity search without managing a separate vector database.

Universal Meta Layer

Use a small, consistent meta-layer so any source can map into .sJson:

  • @type — kind of object
  • @context — domain ontology (optional)
  • @source — original datasource
  • @link — related objects
  • @confidence — certainty or trust level
  • @embedding — vector representation (optional)
  • @timestamp — capture time
  • @transform — recompute/update rules

Adapters that emit these fields create a truly portable smart object format.

Why AI Models Love It

  • Consistent cross‑source semantics
  • Self‑contained context and units
  • Embeddings where needed for similarity
  • Less parsing ambiguity for models

Result: more reliable reasoning with fewer tokens and simpler prompts.

Sample Conversion

From Aion notation to .sJson:


user @role: "Analyst"
query: "Weekly signup trend"
@directive(analyze): "time-series"


{
  "user": { "value": "Analyst", "role": true },
  "query": "Weekly signup trend",
  "directives": [{ "name": "analyze", "args": ["time-series"] }]
}

This preserves intent (role, analyze) so downstream components can respond intelligently.

Semantic Metadata Example

The same data, first as plain JSON, then as Structured JSON with semantics:

Plain JSON


{
  "city": "Daytona Beach",
  "temperature": 82
}

Structured JSON (.sJson)


{
  "@type": "WeatherReport",
  "@context": "weather.schema",
  "city": { "@value": "Daytona Beach", "@geo": [29.2108, -81.0228] },
  "temperature": { "@value": 82, "@unit": "F" },
  "@confidence": 0.98
}

Why this makes AI models smarter

  • Type + context — @type and @context tell the model what this object represents, reducing ambiguity.
  • Grounding — @geo anchors the city to coordinates, enabling maps, weather APIs, and retrieval by location.
  • Units + value — @unit clarifies Fahrenheit; @value keeps the numeric form explicit.
  • Trust signals — @confidence lets downstream logic and models weigh results appropriately.
  • Better retrieval — explicit semantics improve vector and symbolic search with fewer prompt tokens.

Result: fewer prompt hacks, more reliable reasoning, and easier integrations across your stack.

How To Implement

  1. Define syntax — Reserve @ keys (@type, @link, @confidence).
  2. Parse & compile — Author in .aion, compile to .sJson deterministically.
  3. Store — Keep editable text; use MessagePack/Protobuf for speed if needed.
  4. Embed — Generate vectors on write for semantic search.

Result: a semantic memory store ready for retrieval‑augmented AI models.

Normative Notes (v0.1)

  • MUST include @type. SHOULD include @id and @context.
  • Use RFC 3339/ISO 8601 (UTC) for @timestamp (e.g., 2025-01-01T12:34:56Z).
  • If using @embedding, include @embedding_model (string) and @embedding_dim (number).
  • @transform references a rule ID or declarative transform — no arbitrary code execution.
  • Keys prefixed with @ are reserved by the spec; domain fields are unprefixed. Parsers MUST ignore unknown keys.
  • Optional: include @spec pointing to the spec URL/version for validation.
  • Prefer https URLs in @source; custom schemes should be documented.
