How To Use It

AionScript pairs human-readable .aion with canonical .sJson so you can author with intent and ship deterministic JSON.

  1. Define the syntax — Reserve @ keys for meta (@type, @link, @confidence), then author in .aion.
  2. Parse and serialize — Parse to an AST, then compile to .sJson for machines.
  3. Store efficiently — Keep text for editing; use MessagePack/Protobuf for speed if needed.
  4. Add AI integrations — Generate embeddings on write to power semantic search.
{
  "@type": "UtilityClass",
  "name": "bg-mint-500",
  "css": "background-color: var(--color-mint-500)",
  "meaning": "applies mint background color",
  "@embedding": [0.23, 0.11, 0.77]
}

Query semantically: “Find utilities related to green tones” — retrieve by similarity, not only string match.

Inspiration

  • JSON-LD — @context and linked data semantics
  • GraphQL AST — structurally rich queries
  • Protobuf/Avro — type-safe serialization
  • SQLite + pgvector — vector-aware storage

Naming

We call the language AionScript (.aion) and the canonical machine format Structured JSON (.sJson).

Adapters & Data Sources

Bring external data into Aion with structure and meaning.

Files

CSV → array of @type:"Record"; Excel adds @sheet/@column; PDF yields text with @page/@bbox.

Knowledge

Store @embedding inside .sJson (DocumentChunk) to power semantic search without a separate vector DB.

Streaming

Emit typed events like SensorReading with @unit and @timestamp.

Examples

DocumentChunk with Embedding

{
  "@type": "DocumentChunk",
  "text": "Tailwind’s dark mode now uses @custom-variant",
  "@embedding": [0.02, -0.33, 0.81],
  "@source": "tailwindcss.com/docs/dark-mode"
}

SensorReading Event

{
  "@type": "SensorReading",
  "sensor": "humidity-01",
  "value": 56.3,
  "@unit": "%",
  "@timestamp": 1738859821
}

CLI: GitHub → sJson (Prototype)

Convert any GitHub HTML URL into sJson for AI ingestion. Requires Node.js 18+.

# Usage
node tools/github-to-sjson.mjs \
  https://github.com/owner/repo/blob/SHA/path/to/file.html \
  > out.sjson

# Or with a raw URL
node tools/github-to-sjson.mjs \
  https://raw.githubusercontent.com/owner/repo/SHA/path/to/file.html \
  > out.sjson

The prototype extracts headings, links, and text chunks; it emits a @type: HtmlDocument with DocumentChunk entries, @source, and @timestamp. It does not compute embeddings (you can add them in your pipeline).

Universal Meta Layer

Define a minimal, consistent set of meta keys so any adapter can produce smart objects your whole stack understands:

  • @type — what kind of thing it is
  • @context — the domain ontology (optional)
  • @source — original datasource
  • @link — related objects
  • @confidence — certainty or trust level
  • @embedding — vector representation (optional)
  • @timestamp — when it was captured
  • @transform — rules for re-computing or updating

If every adapter outputs these fields, the rest of your system doesn’t care where the data came from.

Why AionScript

1) Solves the "dumb data" problem

Traditional formats encode structure, not meaning. By embedding semantics (type, context, links, confidence, embeddings), data becomes interpretable knowledge so models don’t have to guess.

2) Unifies fragmented systems

APIs use JSON, databases use tables, AI models use embeddings, humans use text. AionScript provides one layer they can all share without fragile translation.

3) Blends symbolic + neural reasoning

Combine explicit structure with vectors via @embedding/@confidence for hybrid reasoning — rules and probabilities together.

4) Makes private/local AI practical

Store, search, and interpret your own data for LLMs offline or on-prem, with consistent semantics and contracts.

5) Ecosystem potential

A clear spec unlocks SDKs, adapters, visualization, knowledge-graph tooling, and vector-aware stores — a foundation like JSON/YAML/GraphQL.

6) Philosophically elegant

Data knows itself: @type (what), @source (where), @confidence (how sure), @link (relations), @context (meaning), @embedding (vector voice).

7) Toward self-describing systems

Files and APIs that explain their purpose to AI enable interoperable, self-organizing intelligence.
